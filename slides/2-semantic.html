header

Элемент header представляет собой набор предварительных данных и навигационных средств... Обратите внимание: предполагается, что элемент header должен содержать заголовок раздела (элементы h1–h6 или hgroup), но это не обязательно. Элемент header может также использоваться для обрамления списка содержимого раздела, формы поиска или любых необходимых логотипов

header

  • Заголовок сайта
  • Заголовок статьи
  • Комментарий

nav

  • Панель навигации
  • Может содержать заголовок
  • Может быть несколько на странице
  • Не переусердствуйте

footer

Элемент footer используется для создания «футера» для расположенного уровнем выше структурного элемента или элемента группировки контента. Элемент footer обычно содержит сведения о соответствующем разделе — информацию об авторе, ссылки на похожие документы, сведения об авторских правах и т. п.

article

Элемент article представляет собой самодостаточный компонент страницы, документа, приложения, сайта — в принципе, автономный и предназначенный для многократного использования и свободного распространения, например, при синдикации. Это может быть публикация на форуме, статья в журнале или газете, запись в блоге, комментарий пользователя, интерактивный виджет и любой другой независимый элемент содержимого.

article

  • Предполагает наличие заголовка
  • Может быть несколько на странице
  • Может быть вложен в другой article

section

Элемент section используется либо как способ разделения страницы на несколько тематических областей, либо как способ разделения статьи на разделы.

aside

Элемент aside используется для выделения раздела страницы, который косвенно связан с окружающим контентом, но который должен располагаться отдельно. В типографике такие разделы обычно оформляются в виде врезок. Этот элемент может использоваться для создания типографских эффектов (например, врезок и боковых панелей), для рекламы, для групп элементов <nav> и для другого контента, который должен располагаться отдельно от основного.

aria-*

WAI-ARIA (Web Accessibility Initiative’s Accessible Rich Internet Applications) — отдельная спецификация, которая «заполняет дыры» в HTML 5 для повышения доступности приложений и веб-страниц.

Введение в WAI-ARIA

aria-role

<header role=banner>...</header>
<aside role=navigation>...</aside>
<section role=content>...</section>

Microdata

Микроданные (microdata) позволяют добавлять семантику содержимому, не используя дополнительные элементы HTML.

Microdata

  • itemscope
  • itemtype
  • itemprop
  • itemref
  • itemid
<div itemscope itemtype="http://schema.org/Organization">
  <span itemprop="name">Яндекс</span>
  Контакты:
  <div itemprop="address"
    itemscope itemtype="http://schema.org/PostalAddress">
    Адрес:
    <span itemprop="streetAddress">Льва Толстого, 16</span>
    <span itemprop="addressLocality">Москва</span>,
  </div>
</div>
